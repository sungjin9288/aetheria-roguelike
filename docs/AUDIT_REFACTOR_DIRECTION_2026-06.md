> 핵심 문제의식: **"로그라이크인데 초반이 너무 쉬워서 플레이어가 떠난다."**

## ✅ 확정된 방향 (Decision Log · 2026-06-24)

**난이도 다이얼: B(균형 로그라이크) + 초반은 단단하게 ("B+")**
Slay the Spire를 기준으로 하되, *초반이 너무 쉽지 않도록* 초반 쪽 다이얼을 한 칸 더 민다. C(가혹)는 한국 모바일 캐주얼 리텐션을 깎으므로 채택하지 않음. 죽음은 항상 **공정 + 메타 완충**을 유지(부당한 죽음 금지).

**측정 가능한 목표 (테스트로 고정할 계약)**

| 지표 | 현재(실측) | B+ 목표 | 비고 |
|------|-----------|---------|------|
| TTFD (방심 시 첫 죽음 가능 시점) | 사실상 없음 | **6~9분** | "방심하면 죽는 순간 1~2회/첫 30분" |
| TTK 비율 (초반, 잡는턴:죽는턴) | ~0.30 | **0.60~0.68** | 잡기 ~5~6턴 / 죽기 ~8~9턴 |
| 회복 희소성 | 무한·완전 | **빠듯한 선택** | 지역당 휴식 제한 + 레벨업 부분회복 + HP 이월 |
| 신입 보호 | 적 약화(HP×0.88) | **약화 제거** | Lv1·첫 2전투만 초미세(HP×0.95), EXP/골드 보너스 제거 |
| 죽음 완충 | 신규엔 사실상 0 | **첫 죽음 영구 보상** | "죽어도 남는다"를 1회차에 학습 |

**구현 단위: Slice 1을 세 축에서 얇게 묶는다** (A 단독 진행 금지 — 부당한 죽음 위험)
> A-2(소모전) + C-1(첫 죽음 메타) + C-2(지역 위험 경고) + B-1(시작 부트) = 응집된 한 덩어리.
> "긴장되는데 공정하고, 죽음이 학습이 되고, 느림이 흥미로워지는" 완결 경험.

**레퍼런스 핵심 차용**: Slay the Spire **캠프파이어**(회복 OR 강화 택1)를 탐험 롤에 이식하면 A-2(소모전)+B(결정)를 동시 해결. Hades **Heat**는 기존 `CHALLENGE_MODIFIERS` 재활용 청사진.

## 0. TL;DR (한 장 요약)

### 진단
초반이 "쉽다"는 건 단일 수치가 아니라 **5개 장치가 겹쳐 만든 결과**다.

1. 신입 보호가 적을 **강제로 약화** (HP ×0.88, ATK ×0.82) — `DifficultyManager.ts:99`
2. 플레이어 공격이 **적 DEF를 전혀 차감하지 않음** — `CombatEngine.ts:48` (`calculateDamage`는 `stats.atk`만 사용)
3. 적 공격은 `max(최소, 적ATK − 내DEF)` **flat 차감** → 초반엔 chip damage — `CombatEngine.ts:1250`
4. 시작 HP 150(직업 보정 후 ~178) vs 초반 몬스터 ATK 15~19 → **죽기까지 9턴, 잡는 데 4턴 (2:1 안전)**
5. 휴식 60골드, **쿨다운 없음·완전 회복** → 소모전(attrition) 부재 — `constants.ts:43`

### 역설
사망은 **레벨 1로 완전 리셋되는 진짜 영구사망**(`handleDefeat`, `CombatEngine.ts:1648`)인데, 위 5개 장치 때문에 **초반엔 죽지 않는다**. → 로그라이크의 핵심 긴장(죽음의 무게)이 한 번도 체감되지 않는다. 그리고 막상 신규 플레이어가 처음 죽을 때(예: Lv14 지역에 잘못 진입)는 "전부 잃음"이 **납득 불가능한 부당함**으로 느껴진다.

### 추가 모순
Slice 23이 "초반을 음미하라"며 레벨업을 **의도적으로 감속**(누적 EXP +34%)했다. 하지만 **난이도·결정 밀도를 안 올린 채 진행만 느리게 하면 = 안전하고 느리고 반복적인 초반** = 이탈의 정확한 레시피.

### 처방 (3축)
- **A. 초반에 이빨을 달기** — 레벨업을 다시 빠르게 하는 게 아니라(그건 계약 테스트로 고정됨), **소모전·결정·텔레그래프된 위협**을 넣는다.
- **B. 느린 분(分)을 결정-밀도로 채우기** — 캐릭터 생성 시 빌드 선택, 유물/스킬 선택을 앞당김. "느림"의 의미를 *지루한 그라인드* → *내 빌드 실험*으로 바꾼다.
- **C. 죽음을 "체감되지만 공정하게"** — 첫 죽음 직후 작은 영구 보상(메타 훅), 지역 위험도 텔레그래프로 "부당한 죽음" 차단.

### 리팩토링 Top 3 (디벨롭과 병행)
1. **`constants.ts`의 `BALANCE: any` / `CONSTANTS: any` 타입화** — 모든 매직넘버가 무타입으로 흐름 (최고 가성비 타입 수정)
2. **555개 `cycle-*.test.js` → 도메인별 통합** (유지보수 부채)
3. **CLAUDE.md 문서 드리프트 수정** — "TypeScript 미사용"이라 적혀있으나 실제는 `.ts`/`.tsx` 전면 사용

**가장 아픈 지점**: 게임의 모든 밸런스 수치를 담은 `constants.ts`가

### 1.5 ⚠️ 잠재 버그 (검증 필요, 밸런스 직결)

**플레이어 출력 데미지에 적 DEF가 반영되지 않는다.**
`attack()`에서 `armor_pen` 유물 처리로 `effectiveDef = stats.def × (1−val)`를 만들지만(`CombatEngine.ts:510`), 정작 `calculateDamage`(`:48`)는 **`stats.atk`만 읽고 `def`를 무시**한다.
- 결과 1: 적의 모든 `def` 수치가 **플레이어 일반/스킬 공격에 무효**. 몬스터·보스의 방어력 설계가 사실상 작동 안 함.
- 결과 2: `armor_pen`("방어 무시") 유물이 **플레이어 출력에 아무 효과 없음** (게다가 줄이는 대상이 적 DEF가 아니라 플레이어 자신의 `stats.def`로 보임 → 이중 의심).

> **확정 판정 (2026-06-24): 버그.** `attack()` 본문(`CombatEngine.ts:461~512`) 어디에서도 `enemy.def`를 참조하지 않음. `armor_pen`은 *플레이어 자신*의 `stats.def`를 줄이고 그마저 `calculateDamage`가 무시 → **완전 무효**.
>
> **✅ 해소 (PR #3, 2026-06-25): 비율 경감 + 형식적 def 재튜닝 동반 수정.**
> Phase 0에서 보류했던 항목. 조사 중 **2차 결함** 발견 — `spawnEnemy`가 def 필드를 *아예 생성하지 않아*(profile의 `hp/atk/def` 리터럴도 dead) 실전 `enemy.def`는 항상 `undefined`였다. 따라서 수식만 고치면 no-op.
> 최종 수정:
> 1. `spawnEnemy`에 def 곡선 추가 — `def = MONSTER_DEF_BASE(3) + level×MONSTER_DEF_PER_LEVEL(1) + depth×PER_DEPTH(1)`, `profile.defMult` 가중 지원(탱키 아키타입용 후속 레버).
> 2. `CombatEngine.mitigateByEnemyDef(raw, def, relics)` — 비율 경감 `max(1, floor(raw×K/(K+effDef)))`, `K=ENEMY_DEF_K(100)`. `calculateDamage`는 순수 roll로 유지(경감은 attack/skill의 finalDamage 후처리 1회).
> 3. `armor_pen`("방어 무시") 유물이 effDef를 실제로 감소 → **dead relic 부활**.
> 결과 곡선(`mid-game-ttk-bands` def-aware 모델 검증): 초반(Lv≤6 def 4~9) ~4~8% 경감(무시 가능, A-1/A-4 초반 레버 보존), 중후반 Lv30 TTK 5→7 / Lv50 5→8 — **firmer하되 밴드(2~9턴) 내**, TTD/TTK ≥1.3 유지.
> 전 몬스터 개별 def 리터럴 재튜닝은 불필요(formula가 균일 대체). 아키타입별 `defMult` 가미는 후속 polish(hpMult 높은 탱크에 곱하면 TTK>9 위험이라 개별 검증 필요).

## 2. 초반 난이도 정밀 진단 (코드 실측)

### 2.1 첫 전투 실측 (모험가 · 고요한 숲 · 슬라임)

```
플레이어 유효 ATK ≈ 14 (기본 12 + 단검 5 × ONE_HAND_ATK_RATIO 0.44)
플레이어 HP ≈ 178 (직업 보정 후)

슬라임 HP = (70 + 1×32) × 0.8(슬라임 hpMult) × 0.88(신입보호) ≈ 72
슬라임 ATK ≈ 19 × 0.82(신입보호) ≈ 15

플레이어 데미지/타 = floor(14 × (0.9~1.1)) ≈ 12~15 (강타 1.5배 → 18~20)
  → 잡는 데 4턴

슬라임 데미지/타 = max(최소, 15 − 내DEF) ≈ 9~14
  → 죽기까지 13~19턴

위험도: 잡는 4턴 vs 죽는 13턴 ≈ 3:1 플레이어 우위 (신입보호 미적용 시에도 ~2:1)
```

> 결론: **초보가 의도적으로 죽으려 해도 첫 여러 전투는 죽기 어렵다.** 텍스트 턴제 특성상 매 턴 직접 클릭하므로, "긴장 없는 클릭 4번"이 반복된다.

### 2.3 DifficultyManager의 철학적 문제 (로그라이크 안티패턴)

`DifficultyManager.ts`는 최근 20전투 성과로 적을 ±15% **고무줄(rubber-band)** 조정한다.
- 잘하면 "압도"(+15%), 보통이면 "균형"(+2%)으로 **항상 편안한 대역에 묶어둔다**.
- 신입(Lv≤3, <5전투)은 **강제로 더 쉽게** + EXP/골드 보너스.

> 로그라이크의 재미는 **일관되게 가혹한 난이도를 빌드 숙련으로 극복**하는 데서 온다. 고무줄은 그 보상 구조를 무너뜨린다 — "내가 강해진 게 아니라 게임이 봐준 것"이 되기 때문. 적응형 난이도는 액션 게임에는 맞지만 **로그라이크 정체성과는 충돌**한다.

### 2.4 핵심 통찰: "느림"과 "쉬움"은 다른 문제다

Slice 23 주석은 실기기 피드백 *"초반 레벨업이 너무 빠르면 안 된다"*에서 출발했다. 방향은 옳았다. 하지만 **진행만 느리게 하고 난이도·결정 밀도를 안 올렸다.** 결과:

```
빠르고 안전함  → 가벼운 도파민 (예전)
느리고 안전함  → 지루함 (지금) ← 이탈
느리고 위험함  → 긴장과 학습 (목표) ← 로그라이크
```

"음미할 시간"은 *위협 속에서 흥미로운 결정을 내리는 시간*이어야지, *못 죽이는 적을 클릭하는 시간*이 아니다.

## 4. 디벨롭 방향성 (구체적 처방)

### 축 A — 초반에 이빨 달기 (난이도)

> 원칙: 레벨업 속도는 건드리지 않는다(계약 테스트 고정). 대신 **소모전·결정·위협**을 추가한다.

**A-1. 신입 보호 재설계 (최우선)**
- 현재: Lv≤3 적을 약화 → 제거 또는 **반전**.
- 권장: 적을 약화하는 대신 **명확한 텔레그래프 + 관대한 회복**으로 학습 지원. (적은 정상 강도, 단 "다음 턴 강공 예고"를 1~2전투 강조 노출)
- 만약 보호 유지 시: **Lv1·2전투로 축소**하고 EXP/골드 보너스는 제거(공정성).

**A-2. 소모전(attrition) 도입 — 가장 큰 레버**
- 휴식: 무한·완전회복 → **지역 진입당 N회 제한** 또는 **쿨다운**. (Slay the Spire의 캠프파이어 모델)
- 레벨업 시 HP/MP 완전회복(`applyExpGain`) → **부분 회복**으로.
- 효과: "다음 전투를 풀피로 못 맞이할 수 있다"는 단 하나의 사실이 모든 전투에 긴장을 부여.

**A-3. 적 DEF 데미지식 반영 (1.5 버그 확정 시)**
- `calculateDamage`에 적 DEF 경감 추가. **flat 차감은 스케일 붕괴** → 비율 경감 권장:
  `mitigated = dmg × 100 / (100 + enemyDef)` (game-balancing 가이드 공식)
- 효과: "맞는 무기·속성을 골랐는가"가 초반부터 의미. 빌드 결정에 무게.

**A-4. 초반 엘리트/위험 노드**
- 고요한 숲에 **고위험 엘리트**(거대/고대 prefix, `MONSTER_PREFIXES` 이미 존재) 1체 — 잡으면 유물 확정.
- StS 엘리트처럼 **선택적 위험/보상**. 초보에게 "이건 위험하다"는 첫 학습.

### 축 B — 느린 분을 결정으로 채우기

**B-1. 캐릭터 생성 시 시작 부트(boon) 선택** — Hades 거울 / StS Neow.
- 시작 유물 1개 또는 스킬 분기 1개를 *0분에* 선택. → 첫 50킬이 "내 빌드 검증"이 됨.

**B-2. 첫 유물 노출 앞당기기** — `FIRST_RELIC_PITY_EXPLORES` 6 → 3.
- 빌드 결정이 초반 학습 구간 안으로 들어옴.

**B-3. 텔레그래프 활용** — `predictEnemyNextAction`(이미 구현)을 초반 UI에서 강조하고, 무시 시 명확히 아프게.

### 축 C — 죽음을 체감되지만 공정하게

**C-1. 첫 죽음 = 메타 튜토리얼**
- 첫 사망 직후 작은 **영구 보너스**(예: prestige bonusAtk +1, 또는 메타 통화 첫 지급). → "죽어도 남는 게 있다"를 *경험으로* 학습 (Rogue Legacy/Hades).
- 현재 메타(`PRESTIGE_*`)는 마왕 격파 뒤에야 열려 신규엔 너무 멀다 → **초반 메타 훅 신설**.

**C-2. 지역 위험도 텔레그래프**
- 권장 레벨 대비 높은 지역 진입 시 **명확한 경고**(이미 `getMapLevel` 존재 → 경고 강화). 완전 리셋 페널티가 있는 한, "모르고 들어가 한 방"은 부당함으로 직결.

**C-3. 챌린지/Heat 노출** — `CHALLENGE_MODIFIERS`(반피런/무일푼/엘리트런 등, `constants.ts:194`)는 *이미 정의되어 있다*. Hades Heat처럼 **선택적 난이도 ante**로 노출 → 고수에게 일관된 도전·보상.

### 축 D — DifficultyManager 철학 전환
- 고무줄의 **하한(연패 시 구제)**은 유지(이탈 방지), **상한(편안함 천장)은 제거**.
- "균형 대역 묶기" → "숙련하면 더 어렵고 더 보상" + "원하면 챌린지로 더 어렵게". 적응형이 *플레이어 의지*를 대체하지 않게.
