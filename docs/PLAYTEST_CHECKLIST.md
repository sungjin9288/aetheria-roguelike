# Playtest Checklist

Aetheria Roguelike 플레이 검증용 체크리스트입니다.

목표:
- 메타 시스템이 실제 플레이 루프에서 끊기지 않는지 확인
- 모바일 실기기에서 터치/레이아웃/저장 동작까지 확인
- 스토어 제출 전 회귀 포인트를 짧은 루틴으로 반복 가능하게 유지

## 0. RC-1 운영 규칙

- 이 단계에서는 새 기능을 추가하지 않는다
- 실기기 QA에서 나온 `P0 / P1`만 수정한다
- 수정 후에는 브라우저/네이티브 검증을 다시 돌린다

분류 기준:

- `P0`: 크래시, 저장 꼬임, 버튼 미동작, 오버레이 겹침, 제출 차단 이슈
- `P1`: 가독성, hit area, 스크롤 튐, 작은 레이아웃 문제
- `P2`: 출시 후 처리 가능한 연출/톤 조정

실행 순서:

1. 아래 1번 사전 준비와 자동 smoke를 모두 통과시킨다
2. iPhone 신규 세이브 5분 루틴을 먼저 수행한다
3. 재료를 보유한 QA 세이브로 iPhone 정비 2분 루틴을 수행한다
4. Android에서 같은 5분 + 2분 루틴을 수행한다
5. 이슈를 `P0 / P1 / P2`로 분류한다
6. `P0 / P1`만 수정하고 같은 체크리스트를 다시 반복한다

## 1. 사전 준비

- [ ] 최신 코드 기준으로 실행 중인지 확인
- [ ] `npm run lint` 통과 확인
- [ ] `npm run build` 통과 확인
- [ ] 웹 또는 실기기 빌드가 최신 번들로 sync 되어 있는지 확인
- [ ] 테스트 계정/세이브를 새로 시작할지, 기존 세이브를 이어받을지 결정

### 자동 smoke

- [ ] `npm run test:unit` 통과
- [ ] `npm run test:smoke` 통과
- [ ] `./scripts/local-playtest.sh` 통과

자동 smoke는 다음을 확인합니다.

- 신규 런 시작
- `status` 명령의 `레벨 / 생명 / 기력 / 골드 / 위치` 표시
- `move 고요한 숲`
- 탐험 중 전투 진입
- 전투 승리와 전투 결과 카드
- 이벤트 처리 경로
- `stats`, `system` 탭 전환
- 데스크톱/모바일 뷰포트 브라우저 회귀

참고:
- 로컬 `vite preview`에서는 `/api/ai-proxy`가 없으므로 AI 이벤트는 폴백 경로를 허용합니다.
- 이벤트가 확률상 오래 안 뜰 때는 smoke 전용 테스트 훅으로 이벤트 처리 UI를 한 번 강제 검증합니다. 실제 플레이 확률은 변경하지 않습니다.

- [ ] 인트로에서 이름 입력 후 정상 시작된다
- [ ] 성별 선택 후 멈추지 않고 게임으로 진입한다
- [ ] 초기 직업이 `모험가`로 시작한다
- [ ] 부트 화면에 내부 단계명 대신 `모험을 준비하고 있습니다`가 표시된다
- [ ] 첫 진입 시 생명/기력/장비/소모품 기본값이 정상이다
- [ ] 첫 플레이 기록이 `첫 여정 / 첫 기술 / 첫 유물` 순서로 시작하며 이전 초기화 안내가 섞이지 않는다

- [ ] 일반 공격/스킬/도주가 정상 동작한다
- [ ] 전투 승리 후 경험/골드/전리품 요약 카드가 정상 노출된다

## 5. 유물 시스템

### 획득/선택

- [ ] 탐색 중 유물 3지선다 오버레이가 정상 노출된다
- [ ] 유물 선택 시 즉시 런 상태에 반영된다
- [ ] 유물 건너뛰기가 정상 동작한다

### 전투형 유물

- [ ] `피의 갈증` 치명타 시 기력 회복이 발생한다
- [ ] `강철 의지`가 적 강타를 막는 경우 로그와 피해가 정상 반영된다
- [ ] `불사의 의지`가 전투당 1회만 발동한다
- [ ] `허공의 심장`이 런당 1회만 발동하고 다음 공격 강화가 적용된다
- [ ] `비전 서지`가 턴마다 기력을 회복한다
- [ ] `연격의 반지`가 연속 공격 후 추가 피해를 준다
- [ ] `죽음의 낙인`이 상태이상 기반 추가 피해를 강화한다

### 전투 시작형 유물

- [ ] `재생 코어`가 전투 시작 시 회복된다
- [ ] `저주받은 반지`가 전투 시작 시 자가 피해를 주고 공격력은 증가한다
- [ ] `혼돈의 보석`이 전투 시작 시 공격력 또는 방어력 버프를 부여한다

### 탐색/보상형 유물

- [ ] `고대 지도` 보유 시 이벤트 체감 빈도가 증가한다
- [ ] `허공의 눈` 보유 시 보스 맵에서 보스 조우 빈도가 증가한다
- [ ] `행운의 동전` 보유 시 드롭 빈도가 증가한다
- [ ] `허공의 눈` 보유 시 보스 드롭 보너스가 적용된다

- [ ] 보스 처치 마일스톤 칭호와 환생 칭호도 함께 표시된다

- [ ] 업적 패널의 제목/설명/보상이 실제 DB 데이터와 일치한다

- [ ] 업적 보상 수령 후 중복 수령이 막힌다
- [ ] 업적 보상 아이템/골드가 실제 인벤토리와 재화에 반영된다
- [ ] 레벨/킬/사망/보스/누적 골드 업적이 각각 정상 해금된다

## 8. 퀘스트/전직/성장

- [ ] 레벨 퀘스트 진행도가 전투 외 성장에서도 반영된다
- [ ] 레벨 5 달성 후 1차 전직이 가능하다
- [ ] 전직 후 생명/기력/기술 구성이 정상 갱신된다
- [ ] 전직 선택에서 `현재 직업 / 능력 경향 / 대표 기술 / 필요 레벨`을 영문 해석 없이 비교할 수 있다
- [ ] 기술 화면에서 `현재 기술 / 기력 / 위력 / 추가 효과 / 다음 전직 기술`을 한 흐름으로 읽을 수 있다
- [ ] 기술 성장은 처음 한 번 무료로 선택되고, 이후 변경은 안전한 지역에서 골드를 사용한다
- [ ] 현상수배/일반 퀘스트 수락/완료/보상 수령이 정상 동작한다
- [ ] 신규 세이브의 첫 이야기 임무가 자동 배정되어 별도 수락 없이 출발할 수 있다
- [ ] 이번 원정의 집중 임무는 최대 3개이며 출정 준비, 지도 표식, 필드 추적기가 같은 임무를 가리킨다
- [ ] `양손 파쇄 / 쌍수 연격 / 방패 요새 / 비전 공명 / 탐험 발견` 빌드 퀘스트가 성향과 맞게 추천된다
- [ ] 전투 승리 누적과 탐험 발견 누적이 빌드 유도 퀘스트 진행도에 반영된다
- [ ] 성향과 맞는 퀘스트는 `성향 추천` 또는 `호응` 배지로 읽힌다

## 9. 상점/인벤토리

- [ ] 상점 제목과 할인 영역이 `마을 상점 / 오늘의 할인 / 이번 주 특별 상품`으로 읽힌다
- [ ] 모든 구매·판매·강화 비용이 `골드`로 표시되고 `CR / G`가 섞이지 않는다
- [ ] 상점에서 `골드 부족 / 직업 제한 / 비교 변화` 상태가 한눈에 읽힌다
- [ ] 장비 비교가 `공격력 / 방어력 / 치명타 / 기력`으로 읽히고 `ATK / DEF / CRIT / MP`가 섞이지 않는다
- [ ] 장비 화면의 공격력·방어력·강화 재료와 주무기·방어구·보조 장비가 390px에서 겹치거나 잘리지 않는다
- [ ] 골드나 재료가 부족해도 `강화 보기`는 열리고 부족 사유가 표시되며, 실제 `강화 시도`만 비활성화된다
- [ ] `강화 보기`에서 현재/다음 능력치, 성공률, 골드·재료 비용, 실패 결과를 소비 전에 비교할 수 있다
- [ ] 강화 창을 취소하면 골드와 재료가 유지되고, `강화 시도`를 누를 때만 표시된 비용이 소비된다
- [ ] 강화 성공 시 주 능력치가 미리보기와 같이 최소 1 오르고, 실패 시에는 안내된 손실 규칙과 일치한다
- [ ] 제작 결과의 그림, 등급·종류, 주 능력치, 현재 장비 대비, 착용 조건, 재료가 제작 전에 보인다
- [ ] 합성의 성공률, 보호 자산, 실패 결과, 결과 후보 그림과 능력치가 실행 가능 여부와 관계없이 보인다
- [ ] 직업 제한 장비는 구매/장착 제한이 정상 동작한다
- [ ] 판매 확인 플로우가 정상 동작한다
- [ ] 퀵슬롯에 등록한 소모품이 실제 인벤토리 수량과 연동된다
- [ ] 마지막 소모품 사용 후 퀵슬롯이 비정상 재사용되지 않는다

## 10. 저장/복원

## 11. 모바일 실기기

### 신규 세이브 5분 루틴

#### iPhone

테스트는 신규 세이브로 시작합니다. 각 구간에서 막히거나 같은 설명을 다시 읽은 순간을 바로 기록합니다.

1. `0:00-0:45` 이름과 외형을 선택해 시작한다. `도전 설정 (선택)`을 펼치지 않아도 시작할 수 있고, 플레이 기록이 `첫 여정 / 첫 기술 / 첫 유물`로 시작하는지 확인한다. 첫 화면에서 다음 행동 하나를 3초 안에 고른다.
2. `0:45-1:30` `임무`를 열어 첫 이야기 임무가 이미 자동 배정되어 있는지 확인한다. 별도 재수락 없이 최대 3개의 `집중 임무`가 출정 준비에 보이고, 지도에서 현재 위치·추천 목적지·임무 표식·경로·`미탐험 / 탐험 중 / 완료` 수치를 한눈에 읽을 수 있는지 확인한다.
3. `1:30-2:45` 추천 지역으로 이동해 `탐험`으로 첫 전투에 진입한다. `적의 행동 / 대응 / 기회`를 보고 일반 공격과 기술의 차이, 적의 다음 행동, 생명 변화를 읽는다. 첫 일반 전투가 대략 4~6턴 안에 끝나는지 확인하고, 기력이 부족한 기술은 이유와 함께 막히는지 확인한다.
4. `2:45-3:45` 전투 결과의 `상태 / 보상 / 다음 행동`과 집중 임무 진행을 확인한 뒤 마을로 정상 귀환한다. 귀환 정리에서 원정 성과와 귀환 이유가 실제 결과와 맞고, `보상 받기 / 생명 회복 / 추천 장비 확인 / 가방 정리 / 제작 / 다음 임무` 중 하나만 다음 행동으로 강조되는지 확인한다.
5. `3:45-4:30` 강조된 정비 행동을 한 번 실행한다. 첫 안전 귀환 이야기가 한 번만 나오고, 한 번의 전투·임무 보상으로 두 단계 이상 급격히 레벨이 오르지 않으며 레벨업과 장비 변화의 이득을 이해할 수 있는지 확인한다.
6. `4:30-5:00` `장비`, `가방`, `지도`, `설정`을 차례로 열어 가로 밀림, 작은 글자, 오터치, 캐릭터와 아이템 그림의 이질감을 확인한다. 홈 화면으로 나갔다가 10초 뒤 돌아와 위치, 임무, 장비, 귀환 결과가 유지되는지 확인한다.

통과 기준:
- 처음 보는 사람도 3초 안에 다음 행동 하나를 고를 수 있다.
- 첫 이야기 임무는 자동 배정되고, 집중 임무·지도 표식·필드 추적기가 같은 목적을 보여 준다.
- 첫 일반 전투가 지루하게 늘어지거나 한두 번의 입력으로 끝나지 않는다.
- 임무 한두 개로 성장 구간을 건너뛰지 않고, 레벨업의 효과는 눈에 보인다.
- 전투 결과와 정상 귀환 정리에서 얻은 보상과 다음 행동 하나가 한 번에 이해된다.
- 첫 안전 귀환 이야기는 흐름을 막지 않고 한 번만 표시된다.
- 첫 이동·첫 방문·첫 승리 기록에 `EXP / Gold / HP / MP / Lv.`, 잘못된 장소 조사, 괄호로 감싼 칭호가 섞이지 않는다.

- 상점과 장비·가방에서 화폐와 능력치 이름이 바뀌지 않고 가격·비교 변화·장착 슬롯을 바로 이해할 수 있다.
- 지도, 장비, 아이템, 캐릭터가 서로 다른 디자인 체계처럼 보이지 않는다.
- 주요 버튼이 오터치 없이 눌리고 노치와 홈 제스처 영역에 가리지 않는다.
- 복귀 후 세션이 유지된다.

### 재료 보유 세이브 2분 정비 루틴

강화 재료와 골드, 제작·합성 재료를 보유한 기존 QA 세이브를 사용합니다. 조건이 없는 항목은 실패로 처리하지 않고 준비되지 않은 입력을 정확히 기록합니다.

1. `0:00-0:40` `장비` 또는 `가방`에서 강화 가능한 장비의 `강화 보기`를 연다. 현재/다음 단계, 주 능력치 변화, 성공률, 골드·재료 비용, 실패 결과가 한 화면에서 읽히는지 확인한다.
2. `0:40-1:10` 먼저 취소해 골드와 재료가 그대로인지 확인한다. 다시 열어 `강화 시도`를 누르고 표시된 비용과 실제 소비가 같은지, 성공 또는 실패 결과가 미리 안내된 규칙과 맞는지 확인한다.
3. `1:10-2:00` `제작`에서 결과 그림·등급·종류·주 능력치·현재 장비 대비·착용 조건·재료를 확인하고, `합성`에서 성공률·보호 자산·실패 결과·후보 그림과 능력치를 비교한다. 실행 가능한 항목 하나가 있으면 결과와 소비가 미리보기와 같은지 확인한다.

통과 기준:
- 강화 취소와 창 닫기는 어떤 재화도 소비하지 않는다.
- 강화 실행 전 정보와 실제 비용·결과 규칙이 일치한다.
- 제작·합성 결과의 정체성과 가치가 실행 전에 보이며, 재료가 부족하면 부족한 입력이 구체적으로 표시된다.
- 강화·제작·합성 화면이 귀환 후 다음 행동을 가리거나 막지 않는다.

#### Android

1. iPhone과 같은 신규 세이브 5분 루틴과 재료 보유 세이브 2분 정비 루틴을 수행한다.
