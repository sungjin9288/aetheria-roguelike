# Playtest Checklist

Aetheria Roguelike 플레이 검증용 체크리스트입니다.

목표:
- 메타 시스템이 실제 플레이 루프에서 끊기지 않는지 확인
- 모바일 실기기에서 터치/레이아웃/저장 동작까지 확인
- 스토어 제출 전 회귀 포인트를 짧은 루틴으로 반복 가능하게 유지

## 0. RC-1 운영 규칙

- 이 단계에서는 새 기능을 추가하지 않는다
- 실기기 QA에서 나온 `P0 / P1`만 수정한다
- 수정 후에는 브라우저/네이티브 검증을 다시 돌린다

분류 기준:

- `P0`: 크래시, 저장 꼬임, 버튼 미동작, 오버레이 겹침, 제출 차단 이슈
- `P1`: 가독성, hit area, 스크롤 튐, 작은 레이아웃 문제
- `P2`: 출시 후 처리 가능한 연출/톤 조정

실행 순서:

1. 아래 1번 사전 준비와 자동 smoke를 모두 통과시킨다
2. iPhone 5분 빠른 루틴을 먼저 수행한다
3. Android 5분 빠른 루틴을 수행한다
4. 이슈를 `P0 / P1 / P2`로 분류한다
5. `P0 / P1`만 수정하고 같은 체크리스트를 다시 반복한다

## 1. 사전 준비

- [ ] 최신 코드 기준으로 실행 중인지 확인
- [ ] `npm run lint` 통과 확인
- [ ] `npm run build` 통과 확인
- [ ] 웹 또는 실기기 빌드가 최신 번들로 sync 되어 있는지 확인
- [ ] 테스트 계정/세이브를 새로 시작할지, 기존 세이브를 이어받을지 결정

### 자동 smoke

- [ ] `npm run test:unit` 통과
- [ ] `npm run test:smoke` 통과
- [ ] `./scripts/local-playtest.sh` 통과

자동 smoke는 다음을 확인합니다.

- 신규 런 시작
- `status` 커맨드 처리
- `move 고요한 숲`
- 탐험 중 전투 진입
- 전투 승리와 전투 결과 카드
- 이벤트 처리 경로
- `stats`, `system` 탭 전환
- 데스크톱/모바일 뷰포트 브라우저 회귀

참고:
- 로컬 `vite preview`에서는 `/api/ai-proxy`가 없으므로 AI 이벤트는 폴백 경로를 허용합니다.
- 이벤트가 확률상 오래 안 뜰 때는 smoke 전용 테스트 훅으로 이벤트 처리 UI를 한 번 강제 검증합니다. 실제 플레이 확률은 변경하지 않습니다.

## 2. 시작 플로우

- [ ] 인트로에서 이름 입력 후 정상 시작된다
- [ ] 성별 선택 후 멈추지 않고 게임으로 진입한다
- [ ] 초기 직업이 `모험가`로 시작한다
- [ ] 첫 진입 시 HP/MP/장비/소모품 기본값이 정상이다

## 3. 코어 플레이 루프

- [ ] `move`로 인접 지역 이동이 정상 동작한다
- [ ] `explore`로 전투 또는 이벤트가 정상 진입한다
- [ ] 일반 공격/스킬/도주가 정상 동작한다
- [ ] 전투 승리 후 EXP/골드/전리품 요약 카드가 정상 노출된다
- [ ] 사망 후 런 요약 카드가 정상 노출된다
- [ ] 재시작 후 기본 상태로 복귀한다

## 4. 접두사 시스템

### 몬스터 접두사

- [ ] 접두사 몬스터가 등장할 때 이름/로그가 정상 표시된다
- [ ] 엘리트 접두사 몬스터는 스탯 상승이 체감된다
- [ ] 접두사 몬스터 처치가 일반 퀘스트 진행도에 반영된다
- [ ] 접두사 몬스터 처치 후 전리품 드롭이 정상 처리된다

### 아이템 접두사 및 세트

- [ ] 접두사 아이템 드롭이 실제로 발생한다
- [ ] 접두사가 붙은 장비의 이름/스탯/속성이 정상 반영된다
- [ ] 같은 접두사 장비 2개 이상 착용 시 세트 효과가 상태창에 표시된다
- [ ] 세트 효과 착용/해제 시 ATK/DEF/HP 수치가 즉시 갱신된다

## 5. 유물 시스템

### 획득/선택

- [ ] 탐색 중 유물 3지선다 오버레이가 정상 노출된다
- [ ] 유물 선택 시 즉시 런 상태에 반영된다
- [ ] 유물 건너뛰기가 정상 동작한다

### 전투형 유물

- [ ] `피의 갈증` 치명타 시 MP 회복이 발생한다
- [ ] `강철 의지`가 적 강타를 막는 경우 로그와 피해가 정상 반영된다
- [ ] `불사의 의지`가 전투당 1회만 발동한다
- [ ] `허공의 심장`이 런당 1회만 발동하고 다음 공격 강화가 적용된다
- [ ] `비전 서지`가 턴마다 MP를 회복한다
- [ ] `연격의 반지`가 연속 공격 후 추가 피해를 준다
- [ ] `죽음의 낙인`이 상태이상 기반 추가 피해를 강화한다

### 전투 시작형 유물

- [ ] `재생 코어`가 전투 시작 시 회복된다
- [ ] `저주받은 반지`가 전투 시작 시 자가 피해를 주고 공격력은 증가한다
- [ ] `혼돈의 보석`이 전투 시작 시 ATK 또는 DEF 버프를 부여한다

### 탐색/보상형 유물

- [ ] `고대 지도` 보유 시 이벤트 체감 빈도가 증가한다
- [ ] `허공의 눈` 보유 시 보스 맵에서 보스 조우 빈도가 증가한다
- [ ] `행운의 동전` 보유 시 드롭 빈도가 증가한다
- [ ] `허공의 눈` 보유 시 보스 드롭 보너스가 적용된다

## 6. 칭호 시스템

- [ ] 새 칭호 획득 로그가 내부 ID가 아니라 표시 이름으로 나온다
- [ ] 시스템 탭에서 칭호 목록이 표시 이름으로 나온다
- [ ] 칭호 활성/비활성이 정상 동작한다
- [ ] 리더보드에서 활성 칭호가 표시 이름으로 노출된다
- [ ] 런 요약 카드에서 활성 칭호가 표시 이름으로 노출된다
- [ ] 보스 처치 마일스톤 칭호와 환생 칭호도 함께 표시된다

## 7. 업적 시스템

- [ ] 업적 패널의 제목/설명/보상이 실제 DB 데이터와 일치한다
- [ ] 달성한 업적만 `수령` 버튼이 활성화된다
- [ ] 업적 보상 수령 후 중복 수령이 막힌다
- [ ] 업적 보상 아이템/골드가 실제 인벤토리와 재화에 반영된다
- [ ] 레벨/킬/사망/보스/누적 골드 업적이 각각 정상 해금된다

## 8. 퀘스트/전직/성장

- [ ] 레벨 퀘스트 진행도가 전투 외 성장에서도 반영된다
- [ ] `Lv.5` 달성 후 1차 전직이 가능하다
- [ ] 전직 후 HP/MP/스킬셋이 정상 갱신된다
- [ ] 현상수배/일반 퀘스트 수락/완료/보상 수령이 정상 동작한다
- [ ] `양손 파쇄 / 쌍수 연격 / 방패 요새 / 비전 공명 / 탐험 발견` 빌드 퀘스트가 성향과 맞게 추천된다
- [ ] 전투 승리 누적과 탐험 발견 누적이 빌드 유도 퀘스트 진행도에 반영된다
- [ ] 성향과 맞는 퀘스트는 `성향 추천` 또는 `호응` 배지로 읽힌다

## 9. 상점/인벤토리

- [ ] 상점에서 `골드 부족 / 직업 제한 / 비교 변화` 상태가 한눈에 읽힌다
- [ ] 직업 제한 장비는 구매/장착 제한이 정상 동작한다
- [ ] 판매 확인 플로우가 정상 동작한다
- [ ] 퀵슬롯에 등록한 소모품이 실제 인벤토리 수량과 연동된다
- [ ] 마지막 소모품 사용 후 퀵슬롯이 비정상 재사용되지 않는다

## 10. 저장/복원

- [ ] 앱 재실행 후 세이브가 정상 복원된다
- [ ] 활성 칭호/업적 수령 상태/유물/퀘스트가 유지된다
- [ ] 전투 중 저장 후 복원 시 치명적인 상태 꼬임이 없다

## 11. 모바일 실기기

### 5분 빠른 루틴

#### iPhone

1. 앱 실행 후 첫 화면에서 `Field Log + Status + Field Actions`가 무리 없이 보이고 모바일 4열 액션 그리드에서 `RESET`까지 한 번에 눌리는지 확인
2. `explore` 3회, `move` 1회, `quest`/`inventory` 탭 열기 1회씩 수행
3. 전투 결과가 나오면 `장비 확인` 또는 `공명 확인`을 눌러 인벤토리 스포트라이트가 제대로 열리는지 확인
4. `Archive Dock`를 열고 `inventory -> map -> system` 순서로 전환한 뒤, `shop`/`quest`/`craft`를 열면 도크가 숨는지 확인
5. `system` 탭에서 `QA READOUT`를 복사하고 필요하면 `EXPORT`로 `qa snapshot json`도 저장한다
6. 홈 화면으로 나갔다가 10초 뒤 복귀

통과 기준:
- 로그가 최상단에서 바로 읽히고, 상태/조작이 그 아래 자연스럽게 이어짐
- 4열 액션 그리드와 `RESET` 위치가 첫 화면에서 오터치 없이 눌림
- `Archive Dock`는 idle에서만 보이고 상점/퀘스트/제작 같은 오버레이를 가리지 않음
- 전리품 검토 후 인벤토리 스포트라이트가 자연스럽게 열린다
- 복귀 후 세션이 유지됨

#### Android

1. 앱 실행 후 첫 화면 가시성 확인
2. `explore` 3회, `move` 1회, `shop` 또는 `rest` 1회 수행
3. 전투 결과가 나오면 `장비 확인` 또는 `공명 확인`을 눌러 인벤토리 스포트라이트가 제대로 열리는지 확인
4. `system` 탭에서 `QA READOUT`를 복사하고 필요하면 `EXPORT`로 `qa snapshot json`도 저장한다
5. 시스템 뒤로가기 1회, 다시 진입 후 상태 유지 확인
6. 앱 전환 후 복귀

통과 기준:
- 시스템 뒤로가기가 의도치 않은 종료/초기화를 만들지 않음
- 전리품 검토 후 인벤토리 스포트라이트가 자연스럽게 열린다
- Field Actions / Field Log 스크롤 영역이 흔들리지 않음
- 복귀 후 게임 상태가 유지됨

### 실행 순서

- [ ] `npm run cap:sync`로 최신 웹 번들을 네이티브 프로젝트에 반영했다
- [ ] Android는 `npm run android:debug` 결과 APK를 설치했다
- [ ] iPhone은 Xcode에서 최신 `ios/App` 프로젝트를 실행했다
- [ ] 기존 앱이 설치되어 있다면 새 번들로 덮어쓴 뒤 세이브 마이그레이션 이상이 없는지 확인했다

### 공통

- [ ] 첫 화면에서 `Field Log + Status Strip + 주요 액션`이 한 화면 안에 무리 없이 들어온다
- [ ] `Field Log`, `Status Strip`, `Field Actions`가 위계상 자연스럽게 읽힌다
- [ ] 시작 화면에서 콜사인을 직접 입력하거나 추천안을 골라 진입할 수 있다
- [ ] `Field Log`가 첫 화면에서 이전보다 더 많은 로그를 보여주고, HUD/탭과 겹치지 않는다
- [ ] `Archive Dock`가 idle에서만 보이고, 열었을 때 바텀시트로 자연스럽게 올라온다
- [ ] `Archive Dock`가 `shop`, `quest`, `craft`, `class` 상태를 가리지 않는다
- [ ] `Archive Dock` 탭 전환 흐름이 오터치 없이 읽힌다
- [ ] 모바일 액션 버튼 4열 그리드와 `RESET` 위치가 오터치 없이 눌린다
- [ ] 세로 화면에서 주요 패널이 잘린다거나 겹치지 않는다
- [ ] 탭 전환 터치가 정상 동작한다
- [ ] 스크롤 영역이 의도대로 동작한다
- [ ] 전투 종료 후 별도 오버레이 없이 `Field Log`에 `전투 정리` 한 줄 요약이 자연스럽게 남는다
- [ ] `추천 / 공명 / 업그레이드 / 주목` 배지가 상점과 인벤토리에서 같은 의미로 읽힌다
- [ ] 상점은 상단 닫기 버튼으로 바로 닫을 수 있고, 닫기 위해 맨 아래까지 스크롤할 필요가 없다
- [ ] 상점은 카드 선택 시 카드 내부 `구매`로 이어지고, 하단 고정 구매 바가 없다
- [ ] 백그라운드 전환 후 복귀 시 상태가 유지된다
- [ ] 보스전 진입 시 `진입 메모 / 경고 칩 / 첫 클리어 보상 힌트`가 한눈에 읽힌다
- [ ] 탐험 예보의 `TEMPO` 칩과 분위기 문구가 현재 지역 성격과 맞게 느껴진다
- [ ] 성향 카드의 `보상 초점 / 임무 초점 / 보스 지시문`이 실제 플레이 판단에 도움이 된다

### 입력/조작

- [ ] `explore`, `move`, `shop/rest/class/quest/craft` 주요 버튼이 한 손 조작으로도 눌린다
- [ ] 인벤토리 탭에서 장비/소모품 버튼이 오터치 없이 눌린다
- [ ] 상세 패널을 열고 닫아도 스크롤 위치나 UI 상태가 비정상적으로 튀지 않는다

### iPhone

- [ ] 시작부터 10분 이상 연속 플레이 시 치명적 레이아웃 깨짐이 없다
- [ ] 노치/안전영역에서 UI 잘림이 없다
- [ ] 홈 제스처 영역 근처 버튼이 눌리기 어렵지 않다

### Android

- [ ] 뒤로가기/앱 전환 후 복귀가 안정적이다
- [ ] 저사양 기기에서도 전투/탭 전환이 과도하게 끊기지 않는다
- [ ] 시스템 뒤로가기 입력이 의도치 않게 앱 종료나 상태 초기화를 일으키지 않는다
- [ ] 제조사 키보드에서 입력창/스크롤 영역이 비정상적으로 흔들리지 않는다

### 실패 시 우선 기록할 것

- [ ] 플랫폼 / OS 버전 / 기기명
- [ ] 첫 실행인지, 기존 세이브인지
- [ ] 어느 화면에서 발생했는지 (`idle`, `combat`, `event`, `shop` 등)
- [ ] 재현 순서 3줄 이내
- [ ] 스크린샷 1장 또는 짧은 영상
- [ ] 가능하면 `qa snapshot json` 파일
- [ ] 앱 재실행 후에도 재현되는지
